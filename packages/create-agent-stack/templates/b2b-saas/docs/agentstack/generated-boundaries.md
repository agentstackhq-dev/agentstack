# Generated Boundaries

Product code in `apps`, `packages`, and `convex` is safe to edit.

Generated anchors in `agentstack.config.json` must remain present. Theme anchors include `packages/theme/package.json`, `packages/theme/tokens.json`, and `packages/theme/src/index.ts`; removing or renaming them breaks validation and generated app assumptions.

For a new capability, generate the first set of product anchors instead of hand-creating parallel files:

```bash
agentstack add feature invoices --surfaces web,mobile --backend convex
```

The feature command writes files in the editable product areas and refuses to overwrite existing files. After generation, edit the generated feature anchors and run the normal validation loop.

Framework vendor glue is regenerated by AgentStack. Do not edit generated vendor glue directly; change framework configuration or product code instead.
