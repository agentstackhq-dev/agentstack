The generated app is a lean consumer workspace.

It is not a framework dump. It is a typed app shell with web, mobile, and Convex surfaces, a schema-driven Agentstack config, package scripts that call the installed agentstack binary, and ignored .agentstack/ state for provider evidence and fixtures.

Generated project contents

Typed config

agentstack.config.ts contains app identity, environments, surfaces, services, telemetry policy, and Clerk Billing entitlement keys through defineAgentstackConfig.

Surfaces

apps/web, apps/mobile, and apps/convex contain the runnable app code. Convex functions live under apps/convex/convex, not a root convex/ directory.

Package boundary

The generated app depends on agentstack. Provider glue, diagnostics, evidence checks, and runbooks stay package-owned instead of copied into the app.

Auth and data

The web surface uses Clerk and Convex, including signed-in protected data smoke hooks and stable data-agentstack-* markers for automated checks.

Billing gate

Convex contains the Clerk Billing webhook route and feature.auditLog entitlement gate used by M3 live validation.

Workspace file

pnpm-workspace.yaml is generated alongside package.json so pnpm and TypeScript language servers resolve apps/* consistently.

Safety defaults

Generated .gitignore protects .agentstack/, .env, and .env.*.

Lifecycle scripts

validate, dev, dev:check, preview:sync, preview:up, provider, auth, billing, smoke, and evidence scripts all delegate to the installed agentstack binary.

Provider state

Provider links, evidence bundles, auth fixtures, ledgers, and smoke artifacts are mutable state under ignored .agentstack/ files.

Runnable Workspace Status

The generated app includes one intentionally small vertical slice so validation checks more than file presence. The web app can render local workspace status, Clerk auth state, Convex protected data, and the M3 entitlement gate. Mobile carries matching app anchors for later execution. 

apps/web/src/App.tsx                         Clerk, Convex, protected-data, entitlement UI
apps/web/src/workspaceStatus.ts             local web workspace fixture
apps/convex/convex/workspaceStatus.ts       protected Convex query anchor
apps/mobile/src/App.tsx                     mobile workspace status surface
apps/mobile/src/workspaceStatus.ts          local mobile workspace fixture

App-specific telemetry, domain packages, docs, and skills are not copied into the generated root by default. Agentstack should add those through package-owned commands when a milestone requires them.

Auth, Billing, And Webhook Anchors

The current runtime proof path is intentionally narrow: Clerk auth, Convex protected data, Clerk Billing webhook ingestion, and a feature.auditLog entitlement gate. Provider-specific Billing slugs come from agentstack.config.ts.

Why two template copies exist

The root templates/b2b-saas copy is easy to inspect in the repository. The package-local copy under packages/agentstack/templates/b2b-saas is what the generator can ship in an npm package. Tests enforce byte-for-byte parity. 

diff -rq templates/b2b-saas packages/agentstack/templates/b2b-saas

Script delegation model

ModeHow scripts find the CLIWhy
Installed package Runs agentstack from local node_modules/.bin or PATH. Generated projects should not contain a machine-specific repo path.
Local source testing Run agentstack create --package-spec link:/packages/agentstack. Keeps consumer tests on the public package boundary while M4 packaging remains locked.
Local preflight Run validate, dev:check, and preview:sync. Checks the typed config, local web start path, and ignored local rehearsal state without mutating live providers.
Local server Run dev when you need the web surface. The command is intentionally long-running and should live in its own terminal.
Live preview Run preview:up -- --confirm-live-mutation. Runs package-owned provider bootstrap, link, auth fixture, and preview deploy in order, stopping at explicit provider handoffs.
Billing fixture Run billing:fixture -- subscribe --env preview --entitlement feature.auditLog --confirm-live-mutation. Uses the package-owned M3 subscription flow and browser handoff when Clerk requires a payment method.
