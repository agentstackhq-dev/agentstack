The generated app is a bounded playground.

It is not a full SaaS product yet. It is a minimal, typed, agent-readable shell with a runnable local workspace-status vertical that proves the framework can own project shape, feature anchors, scripts, safety defaults, docs, validation, and one small product path.

Generated project contents

App identity

agentstack.config.json contains app slug/name, environments, surfaces, enabled services, env declarations, and telemetry policy.

Surfaces

Web, mobile, and Convex now include a small local workspace-status path so validation checks more than file presence.

Shared packages

domain, theme, config, ui, telemetry, and runtime packages carry the shared status contract, tokens, unstyled primitives, and telemetry helpers.

Workspace status

The generated local app path reads a shared workspace-status domain model, exposes Convex status data, renders web and mobile status views, and uses unstyled @app/ui primitive metadata.

SaaS spine

packages/domain/src/saas-spine.ts, convex/saasSpine.ts, and docs/agentstack/saas-spine.md define roles, memberships, billing plans, entitlements, Clerk webhooks, and audit events.

Theme contract

packages/theme/tokens.json is the source of truth. @app/theme imports it, and @app/ui uses token roles instead of hardcoded colors.

Feature workflow

agentstack add feature <name> --surfaces web,mobile --backend convex creates coordinated product anchors before surface code diverges.

Event workflow

agentstack add event billing.subscription.updated --journey billing --surfaces web,convex --state plan:string,seatCount:number creates typed telemetry files and generated event docs.

Agent docs

AGENTS.md, docs/agentstack/, and skills/agentstack/ teach agents the local workflow, feature-first generation, and boundaries.

Guidance skill pack

skills/agentstack/SKILL.md gives installable-skill-aware agents deeper workflows while pnpm run skills:inspect verifies version and file presence. There is no MCP dependency.

Safety defaults

Generated .gitignore protects .agentstack/, .env, and .env.*.

Lifecycle scripts

inspect, doctor, and dev help agents read project state, run preflight checks, and enter the local workspace-status flow before provider-facing work.

Scripts

validate, env:inspect, preview:plan, preview:apply, validate:cloud, preview:deploy, prod:prepare, prod:provision, prod:validate, prod:deploy, and observe:timeline delegate to an installed CLI or explicit source-prototype env vars.

Provider env rehearsal

agentstack env set writes local validation values only. sync --env preview refuses missing or invalid values before planning or applying provider env resources; applied sync stores hash-only metadata in .agentstack/local-cloud.json.

Observability inspection

docs/agentstack/observability.md teaches trace, journey, errors, webhook, component, and compare workflows over redacted local JSONL.

Telemetry export

telemetry:export:preview and telemetry:export:production write an OTLP-shaped JSON local export artifact from redacted store query output without configuring network export or a hosted provider.

Preview runbook

docs/agentstack/preview.md documents exact local rehearsal commands, expected output, files written, and non-goals. It states the workflow is not real provider deployment.

Mobile build profiles

apps/mobile/eas.json and apps/mobile/app.config.ts define generated EAS/Expo anchors. Root and mobile package scripts route through agentstack build mobile.

Runnable Workspace Status

The generated app now includes one intentionally small vertical slice. It is local product behavior across shared domain, Convex, web, mobile, and unstyled UI primitives. It does not make preview deploys, production deploys, EAS builds, or provider mutation real.

Why two template copies exist

The root templates/b2b-saas copy is easy to inspect in the repository. The package-local copy under packages/create-agent-stack/templates/b2b-saas is what the generator can ship in an npm package. Tests enforce byte-for-byte parity. 

diff -qr templates/b2b-saas packages/create-agent-stack/templates/b2b-saas

Script delegation model

ModeHow scripts find the CLIWhy
Installed future package Runs agentstack from local node_modules/.bin or PATH. Generated projects should not contain a machine-specific repo path.
Source prototype Set AGENTSTACK_CLI_BIN and AGENTSTACK_TSX_BIN. Lets this local repo prove real CLI behavior before publishing packages.
Lifecycle preflight Run inspect, doctor, then dev before provider-facing work. dev prints next commands only; real web, mobile, Convex, Expo, and provider servers are future work.
Agent guidance Run skills:inspect, then read skills/agentstack/SKILL.md and matching references for the task. The framework gives agents versioned workflows without making MCP part of the product contract.
Local preview deploy rehearsal Run preview:deploy for PLAN deploy preview and preview:deploy:apply for APPLIED deploy preview. Writes .agentstack/deployments/preview.json and telemetry locally without calling provider APIs.
Local provider env resource rehearsal Run env set for local validation values, then sync --env preview --apply and validate --cloud --env preview. Stores redacted provider env resource metadata and valueHash in local-cloud state, never raw env values.
Local production release rehearsal Run prod:prepare, prod:provision, prod:validate, prod:deploy, and their apply variants where available. Plans and applies local production state and artifacts only; prod:deploy:apply requires explicit production confirmation through the script and calls no provider APIs.

Telemetry runtime anchors

Web, mobile, and Convex anchors import generated event definitions and call createAppTelemetry(runtime).event(...). The helper creates typed local envelopes only. Local JSONL remains the source for inspection, and export scripts write an OTLP-shaped JSON local export artifact without sending app events to a hosted provider.
