The generated app is a bounded playground.

It is not a full SaaS product yet. It is a minimal, typed, agent-readable shell that proves the framework can own project shape, feature anchors, scripts, safety defaults, docs, and validation.

Generated project contents

App identity

agentstack.config.json contains app slug/name, environments, surfaces, enabled services, env declarations, and telemetry policy.

Surfaces

Web, mobile, and Convex anchors exist so validation can detect accidental deletion or malformed project shape.

Shared packages

domain, theme, config, ui, telemetry, and runtime anchors establish where future type-safe shared code belongs.

Feature workflow

agentstack add feature <name> --surfaces web,mobile --backend convex creates coordinated product anchors before surface code diverges.

Agent docs

AGENTS.md and docs/agentstack/ teach agents the local workflow, feature-first generation, and boundaries.

Safety defaults

Generated .gitignore protects .agentstack/, .env, and .env.*.

Scripts

validate, env:inspect, sync:preview, validate:cloud, and observe:timeline delegate to an installed CLI or explicit source-prototype env vars.

Why two template copies exist

The root templates/b2b-saas copy is easy to inspect in the repository. The package-local copy under packages/create-agent-stack/templates/b2b-saas is what the generator can ship in an npm package. Tests enforce byte-for-byte parity. 

diff -qr templates/b2b-saas packages/create-agent-stack/templates/b2b-saas
