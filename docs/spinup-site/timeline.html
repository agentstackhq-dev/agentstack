Implementation timeline

The prototype was built as a managed sequence of subagent-owned slices with spec review, code-quality review, and final blocker reviews. The important part is not the number of commits; it is that each layer became mechanically verifiable.

Final verification

  • pnpm install --frozen-lockfile
  • pnpm typecheck
  • pnpm test: 79 tests
  • Template mirror diff: no output
  • Smoke path: create, validate, env inspect, sync plan/apply, cloud validate, observe timeline

Build sequence

Design and plan

Captured the agent-first framework design and the slice-1 implementation plan under docs/superpowers/.

Workspace and core

Established pnpm/TypeScript/Vitest, diagnostics, strict manifest parsing, env graph, and local validation.

Telemetry and local cloud

Added wide events, redaction, JSONL querying, adapter interfaces, and local-cloud state simulation.

CLI and generator

Implemented command router, local commands, generated B2B SaaS template, package-local templates, and executable generator bin.

E2E and docs

Added generated-project workflow coverage and the human-facing README smoke.

Final blocker hardening

Closed review gaps: anchor validation, stale drift reconciliation, CLI package runtime, generated script delegation, env-value safety, and generated secret ignores.

Slice 2 lifecycle

Added environment-scoped service expectations, provider inspect/plan/apply lifecycle, command telemetry timelines, and stronger generated app anchors.

Review themes that changed the code

Detecting is not enough

Cloud stale drift was detected before sync could repair it. The final adapter now unlinks stale selected-environment services during sync.

Generated scripts must not lie

The first wrapper duplicated validation. The final wrapper delegates to the real CLI and avoids embedding local machine paths.

Env values need a real local source

Required custom env declarations now read from a documented, gitignored, schema-validated .agentstack/env-values.json.

Template docs are part of validation

Generated docs, package/workspace files, surface anchors, and non-file cases are all mechanically checked.

Evidence: git history, final test output, and final review notes.
