Implementation timeline

The prototype was built as a managed sequence of subagent-owned slices with spec review, code-quality review, and final blocker reviews. The important part is not the number of commits; it is that each layer became mechanically verifiable.

Final verification

  • pnpm install --frozen-lockfile
  • pnpm typecheck
  • pnpm test
  • Template mirror diff: no output
  • Smoke path: create, add feature, add event, validate, env inspect, sync plan/apply, cloud validate, preview deploy plan/apply, mobile build plan/apply, observe timelines

Build sequence

Design and plan

Captured the agent-first framework design and the slice-1 implementation plan under docs/superpowers/.

Workspace and core

Established pnpm/TypeScript/Vitest, diagnostics, strict manifest parsing, env graph, and local validation.

Telemetry and local cloud

Added wide events, redaction, JSONL querying, adapter interfaces, and local-cloud state simulation.

CLI and generator

Implemented command router, local commands, generated B2B SaaS template, package-local templates, and executable generator bin.

E2E and docs

Added generated-project workflow coverage and the human-facing README smoke.

Final blocker hardening

Closed review gaps: anchor validation, stale drift reconciliation, CLI package runtime, generated script delegation, env-value safety, and generated secret ignores.

Slice 2 lifecycle

Added environment-scoped service expectations, provider inspect/plan/apply lifecycle, command telemetry timelines, and stronger generated app anchors.

Slice 3 feature generation

Added agentstack add feature so agents can create coordinated domain, Convex, web, mobile, telemetry, and docs anchors through one command.

Slice 4 env values

Added agentstack env set so scoped custom env values are written through validated, redacted CLI state instead of manual JSON edits.

Preview deploy rehearsal

Added local-only PLAN deploy preview and APPLIED deploy preview flows, a secret-free .agentstack/deployments/preview.json artifact, and agentstack.deploy.completed telemetry.

Typed app telemetry

Added generated @app/telemetry metadata, typed event definitions, createAppTelemetry envelopes, agentstack add event docs, and the telemetry-generation observe timeline.

Theme and UI foundations

Added agentstack theme validate, structured theme tokens, a JSON-backed typed theme wrapper, unstyled @app/ui primitives, pnpm package-manager pinning, and generated docs for token-role styling.

Mobile build profiles

Added agentstack build mobile, development/preview/production profile normalization, local build artifacts, generated eas.json/app.config.ts anchors, and the mobile-build telemetry journey.

Billing plan generation

Added agentstack add billing-plan so agents can create coordinated billing-plan anchors across domain, Convex, web, mobile, telemetry, docs, and the billing telemetry journey.

Versioned agent guidance

Added skills/agentstack/, docs/agentstack/skills.md, guidance-version validation warnings, and agentstack skills inspect so agents get installable workflows without MCP as a product dependency.

Production release rehearsal

Added generated prod:prepare, prod:provision, prod:validate, and prod:deploy scripts plus release docs for local production rehearsal without real provider API calls.

Review themes that changed the code

Detecting is not enough

Cloud stale drift was detected before sync could repair it. The final adapter now unlinks stale selected-environment services during sync.

Generated scripts must not lie

The first wrapper duplicated validation. The final wrapper delegates to the real CLI and avoids embedding local machine paths.

Env values need a real local source

Required custom env declarations now read from a documented, gitignored, schema-validated .agentstack/env-values.json.

Source secrets must stop locally

Validation and preview deploy commands block raw secret-like values in source, docs, and .env files before local-cloud deploy rehearsal behavior runs, without printing the detected value.

Template docs are part of validation

Generated docs, package/workspace files, surface anchors, and non-file cases are all mechanically checked.

Guidance needs its own contract

Concise AGENTS.md rules are not enough for deep workflows. The generated skill pack is versioned, inspectable, and still portable to agents that do not support installable skills.
