Validation is the agent contract.

The prototype is designed around concrete, normalized diagnostics. A coding agent should learn what is wrong, where it is wrong, why it blocks work, and what command or file repairs it.

The checks

Manifest schema

parseManifest uses a strict Zod schema. Unknown keys and wrong shapes fail as manifest.invalid.

Generated anchors

Required files include root package/workspace files, docs, web/mobile packages, Convex schema, domain/theme, and telemetry anchors. Directories do not count as files.

Custom env values

.agentstack/env-values.json is validated as environment -> surface -> variable -> string. It is ignored by generated .gitignore.

Source secret policy

agentstack validate and the guarded agentstack deploy --env path scan source, docs, and .env files for raw secret-like values while skipping local state, dependency, build, and cache directories.

Theme token contract

agentstack theme validate and the local validation gate check token JSON shape, token value types, malformed JSON, and typed-wrapper drift.

Cloud drift

Local-cloud validation reports missing services and stale linked services. Sync links missing state and unlinks stale state for the selected environment. Preview deploy apply writes a secret-free local artifact.

Mobile build readiness

agentstack build mobile --env preview requires local validation and linked EAS state for that environment before it can plan or apply a local build rehearsal.

Diagnostic anatomy

FAIL cloud.service.stale
Path: preview.clerk
clerk is linked in preview but is not enabled in the manifest.
Fix: Remove preview.clerk from local cloud state or re-enable it,
then run agentstack sync --env preview --apply.
Blocks: validate --cloud
FAIL source.secret.detected
Path: apps/web/src/leak.ts
Potential raw secret found in apps/web/src/leak.ts.
Fix: Move the value to a provider secret store or set local
validation state with agentstack env set.
Blocks: validate, validate --cloud, deploy
PLAN deploy preview
- planned release preview.vercel
- planned release preview.eas

APPLIED deploy preview
- applied release preview.vercel
- applied release preview.eas
PLAN mobile build preview
- planned eas profile preview distribution internal development-client=no

APPLIED mobile build preview
- applied eas profile preview distribution internal development-client=no

Why this matters

Agents waste time when failures are vague. The diagnostic format gives a stable code, a path, a plain-English message, an actionable fix, and the commands it blocks.
