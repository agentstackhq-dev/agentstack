System map

The current framework is intentionally split into small packages. The kernel owns schemas and diagnostics; adapters simulate provider state; telemetry owns event inspection; the CLI composes them; the generator emits a minimal B2B SaaS app.

Package ownership

@agentstack/core

Defines the manifest schema, diagnostics, env graph, env-value parsing, theme contract, local validation, and generated-anchor policy.

@agentstack/adapters

Defines provider adapter interfaces and the filesystem-backed local-cloud adapter used by this slice.

@agentstack/telemetry

Creates wide events, redacts sensitive state, stores JSONL, and runs local queries.

@agentstack/cli

Loads project context, composes validation, runs local-cloud commands, and queries telemetry.

agentstack create

Generates a B2B SaaS project from the template and package-local template mirror.

templates/b2b-saas

The generated app shape: web, mobile, Convex, typed config, root guidance, package scripts, workspace file, and safety ignore rules.

Layering rule

Core packages do not know about the filesystem except through the CLI. Core computes what should be true. The CLI checks the project on disk and passes structured inputs to core/adapters/telemetry.

What is deliberately local

AreaPrototype behaviorCurrent provider boundary
Cloud providers .agentstack/local-cloud.json simulates linked services per environment, and .agentstack/deployments/preview.json records local preview deploy rehearsal output. inspect shows contract-only or per-provider command-plan adapter status with stable pending operation IDs. Provider plan prints redacted pnpm exec clerk, pnpm exec convex, pnpm exec vercel, and pnpm exec eas command shapes without executing provider commands. Explicit live reads inspect Clerk/Convex/Vercel/EAS diagnostics; EAS project-info is proof-only and still refuses readiness. Provider apply executes Convex and Vercel preview deploy only.
Env values .agentstack/env-values.json satisfies required custom env declarations locally. Provider env operations expose surface scope and variable names only. Command plans label values as coming from .agentstack/env-values.json, Clerk Dashboard, or provider-owned env commands; values and hashes stay out of operation IDs, inspect output, and command output. Convex inspect/apply requires CONVEX_DEPLOY_KEY, Clerk inspect requires an authenticated Clerk CLI or CLERK_SECRET_KEY, Vercel plans require VERCEL_TOKEN, and EAS plans require EXPO_TOKEN.
Telemetry .agentstack/events.jsonl stores redacted-queryable wide events, including agentstack.deploy.completed for preview rehearsals. OTel exporter/provider can replace the local store without changing event shape.

Provider orchestration boundary

Boundary invariant

The repo already models provider intent, required tokens, production confirmation, env resource scope, and operation IDs. It intentionally stops before remote mutation. That means local validation and command plans can be reviewed today, while direct provider execution runs only through explicit provider inspect/apply.
