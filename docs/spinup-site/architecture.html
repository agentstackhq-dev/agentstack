System map

The prototype is intentionally split into small packages. The kernel owns schemas and diagnostics; adapters simulate provider state; telemetry owns event inspection; the CLI composes them; the generator emits a minimal B2B SaaS app.

Package ownership

@agentstack/core

Defines the manifest schema, diagnostics, env graph, env-value parsing, local validation, and generated-anchor policy.

@agentstack/adapters

Defines provider adapter interfaces and the filesystem-backed local-cloud adapter used by this slice.

@agentstack/telemetry

Creates wide events, redacts sensitive state, stores JSONL, and runs local queries.

@agentstack/cli

Loads project context, composes validation, runs local-cloud commands, and queries telemetry.

create-agent-stack

Generates a B2B SaaS project from the template and package-local template mirror.

templates/b2b-saas

The generated app shape: web, mobile, Convex, domain, theme, telemetry, docs, scripts, and safety ignore rules.

Layering rule

Core packages do not know about the filesystem except through the CLI. Core computes what should be true. The CLI checks the project on disk and passes structured inputs to core/adapters/telemetry.

What is deliberately local

AreaPrototype behaviorFuture real adapter behavior
Cloud providers .agentstack/local-cloud.json simulates linked services per environment. Convex, Clerk, Vercel, and EAS adapters implement the same validate/sync interface.
Env values .agentstack/env-values.json satisfies required custom env declarations locally. Provider-specific env APIs and secret stores can be wrapped behind the framework.
Telemetry .agentstack/events.jsonl stores redacted-queryable wide events. OTel exporter/provider can replace the local store without changing event shape.
