Wide events make debugging explainable.

The telemetry package models events as high-resolution snapshots of state. The local implementation uses JSONL so agents can inspect journeys without a hosted control plane.

Event shape

createWideEvent("onboarding.step.completed", {
  environment: "preview",
  surface: "web",
  journey: "onboarding",
  correlationId: "corr_onboarding",
  state: {
    step: "workspace",
    email: "person@example.com"
  }
});

What makes it wide

The event includes operational context plus a structured state snapshot. That lets a future timeline view reconstruct what happened without forcing agents to correlate ten narrow log lines by hand.

Query filters

Environment

--env preview scopes incidents to the deployed context.

Surface

--surface web separates web, mobile, convex, cli, and provider surfaces.

Event family

--event billing.* uses wildcard matching for families of events.

Journey

--journey onboarding reconstructs user journeys.

Trace

--trace trace_123 can follow request-level context.

Correlation

--correlation corr_123 groups related actions across surfaces.

Why it matters

Agents need production-debugging visibility, but not raw secrets. The telemetry store redacts sensitive fields before query output, including email/token/key-like fields.
