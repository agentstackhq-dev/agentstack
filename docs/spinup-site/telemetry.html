Wide events make debugging explainable.

The telemetry package models events, spans, journeys, identity, and redaction as provider-neutral local envelopes. This prototype reads redacted local JSONL for inspection and can write an OTLP-shaped JSON local export artifact for handoff without claiming network export, hosted indexing, or provider dashboards.

Runtime primitives

const telemetry = createAppTelemetry(webRuntime).identify({
  actorId: "user_123",
  orgId: "org_123",
  journeyId: "journey_billing_123"
});

telemetry.event(billingSubscriptionUpdatedEvent, { plan: "team", seatCount: 5 });
telemetry.span("convex.billing.applySubscriptionUpdate", { plan: "team" });
telemetry.journey("billing", "subscription-updated", { plan: "team" });
telemetry.redact({ email: "person@example.com", plan: "team" });

What gets created

The runtime helper creates provider-neutral local envelopes for typed events, spans, and journey phases. State is redacted before it appears in generated telemetry envelopes, and no hosted telemetry, network export, provider ingestion, or dashboard link is configured by these primitives.

Local export artifact

pnpm run telemetry:export:preview
pnpm run telemetry:export:production

node scripts/agentstack.mjs observe export \
  --env preview \
  --format otlp-json

Where export fits

Local JSONL remains the source for local inspection. The export command writes an OTLP-shaped JSON local export artifact from redacted store query output so agents can hand off evidence without configuring network export or a hosted provider.

Generated event workflow

node scripts/agentstack.mjs add event billing.subscription.updated \
  --journey billing \
  --surfaces web,convex \
  --state plan:string,seatCount:number

node scripts/agentstack.mjs observe timeline \
  --env development \
  --journey telemetry-generation

What gets created

The command writes a typed file under packages/telemetry/src/events/ and a generated note under docs/agentstack/events/. The command itself is visible as agentstack.event.added in the local JSONL timeline.

Inspection modes

Use query and timeline for discovery, then switch to the mode that answers the incident question. Every mode below reads the same redacted .agentstack/events.jsonl stream in this prototype.

Query filters

Environment

--env preview scopes incidents to the deployed context.

Surface

--surface web separates web, mobile, convex, cli, and provider surfaces.

Event family

--event billing.* uses wildcard matching for families of events.

Journey

--journey onboarding reconstructs user journeys.

Trace

--trace trace_123 can follow request-level context.

Correlation

--correlation corr_123 groups related actions across surfaces.

Component

--component convex:billing.applySubscriptionUpdate isolates a runtime boundary.

Time window

--since 2h keeps agents from debugging stale local history.

Error class

--error-class WebhookSignatureError narrows grouped failures.

Why it matters

Agents need production-debugging visibility, but not raw secrets. The telemetry store redacts sensitive fields before query output, including email/token/key-like fields. Local export artifacts are redacted because they are built from the same redacted store query output. Network export, hosted providers, and retention controls are future adapter work.
