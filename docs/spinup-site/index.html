What we built, why it matters, and how the pieces fit.

Agentstack is an agent-first meta-framework package. The current milestone contract is lean: consumers run agentstack create, get a small app root, and use package-owned commands for validation, provider bootstrap/link/deploy, auth fixtures, Clerk Billing fixtures, smoke checks, and evidence. Some deeper pages on this site preserve earlier prototype slices; use README, docs/README, and the milestone cards as the current source of truth.

The short version

What exists now

A local source package that can generate a lean consumer app, validate typed agentstack.config.ts, run local dev checks, rehearse preview state, execute the live preview happy path, and verify the M3 Clerk Billing entitlement gate through package-owned commands.

What it proves

An agent can start from an empty directory and use one public command surface: agentstack create, generated package scripts, and package-owned provider/auth/billing/evidence flows. The generated app no longer receives copied docs, scripts, skills, root Convex, or framework package internals.

The loop Agentstack is proving

CreateCopy the app template and replace identity tokens.agentstack create
ValidateCheck typed config, app anchors, and package-owned boundary.pnpm run validate
CheckVerify the local web start path without running a long-lived server.pnpm run dev:check
RehearseWrite ignored local preview rehearsal state only.pnpm run preview:sync
PreviewBootstrap/link/deploy live provider preview state behind confirmation.pnpm run preview:up
EvidenceRun smoke and evidence checks from package-owned commands.evidence:check
