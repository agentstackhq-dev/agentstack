What we built, why it matters, and how the pieces fit.

Agentstack is currently a local, executable proof of an agent-first framework contract. It does not call Convex, Clerk, Vercel, or EAS yet. It now proves a generated workspace-status app vertical as well as the command surface, validation model, generated project shape, cloud-drift loop, provider command-plan boundary, and telemetry inspection path bounded to local JSONL inspection and local OTLP-shaped artifact export.

Current repo atlas

Agentstack currently contains a framework workspace, a packageable generator, a mirrored B2B SaaS template, and a generated-project contract that the smoke workflow can create on demand. The framework packages are source TypeScript; the generated app is the shape they create and validate.

The short version

What exists now

A local-first framework kernel with a manifest schema, generated app template, runnable workspace-status surface, CLI command router, validation engine, filesystem-backed local-cloud adapter, JSONL telemetry store, and end-to-end prototype workflow test.

What it proves

An agent can bootstrap a project, run a small cross-surface local app path, validate local structure, initialize simulated provider state, reconcile drift, and inspect redacted telemetry without getting lost in provider dashboards, env state, or hand-written setup steps.
