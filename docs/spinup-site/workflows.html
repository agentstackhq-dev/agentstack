Command journeys

The important outcome is not that individual functions exist. It is that an agent can move through a coherent workflow: create a project, inspect lifecycle state, add a typed feature, validate it, initialize local-cloud state, repair drift, rehearse a preview deploy locally, and inspect telemetry.

Happy path

Source prototype smoke

pnpm install
agentstack create acme-crm --package-spec link:/packages/agentstack
cd acme-crm
pnpm run validate
pnpm run dev:check
pnpm run preview:up -- --confirm-live-mutation

What each command proves

  1. agentstack create copies the lean template, replaces app tokens, and writes a typed agentstack.config.ts.
  2. validate checks the typed config, generated anchors, env contract, and package-owned framework boundary.
  3. dev:check verifies the local web start path without running a long-lived server; dev starts the web surface for local work.
  4. preview:up -- --confirm-live-mutation runs the package-owned live preview sequence and stops at exact provider auth or browser handoffs when needed.

Failure and repair signals

SituationSignalRepair
Generated file missing FAIL template.anchor.missing Restore the generated file or regenerate the app from a clean target directory.
Stale or wrong binary agentstack --help does not show create Follow docs/references/local-quickstart.md to remove the stale symlink and reinstall from this checkout.
Local package dependency resolves to npm ERR_PNPM_NO_MATCHING_VERSION agentstack@0.0.0 Use agentstack create --package-spec link:/packages/agentstack or repair the generated dependency.
Preview rehearsal confused with live mutation preview:sync reports local rehearsal scope Use preview:sync for ignored local state only; use preview:up -- --confirm-live-mutation for live provider work.
Live preview confirmation missing FAIL preview.up.confirmation-required Rerun pnpm run preview:up -- --confirm-live-mutation only when live provider mutation is intended.
Clerk Billing needs a payment source Billing fixture reports the browser SDK handoff Follow docs/references/m3-clerk-billing-fixture.md, then rerun the package-owned subscribe command.
